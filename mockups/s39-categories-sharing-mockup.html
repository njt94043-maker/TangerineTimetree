Takes = recorded performances. When you record yourself playing along (via EAD-10 USB on native, or mic/line-in on web), each recording is saved as a "take". You can have multiple takes per song. Mark your best one — it gets promoted to the stem list so other members can hear it during practice.

Takes are stored as stems with source = 'recorded' and your created_by.

How takes work:
- Record via EAD-10 USB (native practice mode) or selfie-record (future web feature)
- Each recording saved as a take (stem with source='recorded')
- Only visible to the owner by default (your practice history)
- Click "Set Best" → that take gets is_best_take=true and appears in the stems list for everyone
- Changing best take swaps which one is promoted — old best goes back to takes-only
- Other members see it as a regular stem in practice mode (mute/solo/gain controls)
- Only one best take per user per song

Post-recording flow (D-139, D-145):
- 4 options presented immediately when recording stops — no extra navigation
- Discard & Re-take: deletes audio + video, jumps straight back to recording. For bad takes.
- Save & Re-take: keeps this take locally, immediately starts a new recording. Most common workflow — iterate fast.
- Save as Take: keeps locally, returns to takes list. For reviewing later.
- Save & Preview: keeps locally, plays it back immediately so you can hear yourself.
- Mark as Best toggle: OFF by default. When ON, the audio uploads to Supabase storage and becomes the promoted stem visible to all members. Replaces previous best take in cloud. Video is never uploaded.
- Non-best takes stay in IndexedDB — available for local playback/review but don't consume cloud storage.
