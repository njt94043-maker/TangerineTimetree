1 · End-to-end workflow

Every gig follows the same path, from raw capture to released stereo bounces.

E6330 capture → pull-gig.py → build-postprod-rpp.py → open in Reaper → setup-postprod-fx.lua → dial in params → region per song → render

Step-by-step

  1. Sync the gig. On OptiPlex, terminal: python D:/Gigs/pull-gig.py [YYYY-MM-DD]. Defaults to today's date. Pulls 18ch WAVs + per-set set-XXXX.RPP files via Tailscale SSH from E6330.
  2. Build the post-prod project.
    • Single set: python D:/Gigs/build-postprod-rpp.py D:/Gigs/<date>/set-XXXX.RPP
    • Whole gig (multi-set): python D:/Gigs/build-postprod-rpp.py <set1>.RPP <set2>.RPP <set3>.RPP (chronological order; auto-stitches with set boundaries marked, last short set auto-named "Encore")
  3. Open the generated project in Reaper: D:/Gigs/<date>/<date>-whole-gig-postprod.RPP (or <set>-postprod.RPP for single-set).
  4. Run the FX installer. Actions → Show action list → Load… → D:/Gigs/setup-postprod-fx.lua → run. ~50 plugins auto-inserted across channels, buses, and master. Idempotent — safe to re-run.
  5. Dial in per-channel params. Use the tables below — click each channel/bus/master to set the recommended starting values. Save (Ctrl+S) when done so chains persist.
  6. Mark song boundaries. During playback at each song-start, press your bound hotkey (M if you bound insert-named-marker.lua) → type song name → marker drops. Or select a range, then Insert → Region (named).
  7. Render. Path A (shared mix, per-song stereo bounces): File → Render → Region Render Matrix. Path B (per-song unique mixes): python D:/Gigs/split-into-songs.py <postprod.RPP>, then open each generated songs/<name>.RPP independently.
Re-running Both Python scripts and the FX installer are idempotent. You can rerun them after editing the source set RPPs (e.g. if you re-pulled fresh recordings). The FX installer skips plugins already present on a track — only adds missing ones.

4 · James leveling deep-dive

The problem: James moves further from the mic when he sings louder, so his loud parts come through paradoxically QUIETER than his quiet parts. Inverse-square law: doubling distance = -6 dB at the mic, so a 30 cm → 60 cm move = -6 dB, a 30 → 90 = -10 dB. Easy 6-12 dB swings. A single fast comp can't tame this — it's tracking transients, not averages.

The architecture: 3-stage compression

StageWherePluginTime constantJobTarget GR
1Track (Ch 3)ReaCompFAST (10 ms / 80 ms)Catch transient peaks (consonants, plosives).3-5 dB
2 levelerTrack (Ch 3)MJUCjrSLOW vari-mu (recovery slow)Level the macro dynamic range from distance variation.1-2 dB
3VOX BUSDC1A3 + MJUCjrMid (DC1A3 Glue)Final vocal-bus glue + warmth.1-2 dB total

Why MJUCjr specifically

Vari-mu compression has two properties that make it ideal for leveling:

  • Soft, ratio-dependent compression curve — ratio gets gentler at lower input levels, harder at higher levels. Naturally wants to ride averages, not transients.
  • Slow time constants don't sound like "compression" — they sound like a fader rider gently riding levels. No pumping, no "squashed" character.

If still inconsistent after dial-in

Loud parts still quieter than softPush MJUCjr Comp to 40-50%. Recovery still slow.
Vocal sounds squashed / unnaturalDrop MJUCjr Comp to 20-25%. Or switch Recovery to Mid.
"Pumping" between phrasesBus comp (DC1A3 in Glue mode) might be too aggressive. Lower its Input by 2-3 dB so GR is <1 dB.
Transients still poking outReaComp ratio to 4:1, attack to 5 ms. Don't go past that or transients distort.

The "free fader rider" alternative

For songs where automation is worth it (singles, signature tunes), draw volume automation envelope on the James track riding the louder parts UP and quieter parts down to a target level (usually around -8 dBFS RMS). MJUCjr handles 80% of it; automation handles the other 20% surgically. Track Envelope: Volume action, then draw curve.

7 · Render

Path A: shared mix, per-song stereo bounces

Your global mix applies; each region renders to its own stereo file. Best for normal release flow.

  1. Mark song boundaries (Insert → Region or hotkey-bound script).
  2. File → Render.
  3. Source: Master. Bounds: Project regions.
  4. Click Region Render Matrix button.
  5. Tick the Master column for every region you want bounced.
  6. Format: WAV, 24-bit, 48 kHz. (Pre-set in the project's Render config.)
  7. File pattern: $project-$region.wav → output goes to mixdowns/ subfolder.
  8. Render. Each region becomes its own file.

Path B: per-song multitrack projects (unique mixes per song)

When a song needs surgical mix tweaks the global mix can't deliver.

  1. Mark song regions in the post-prod project (same as Path A).
  2. Save project (Ctrl+S).
  3. Terminal: python D:/Gigs/split-into-songs.py D:/Gigs/<date>/<date>-whole-gig-postprod.RPP
  4. Generates D:/Gigs/<date>/songs/<song-slug>.RPP per region. Each opens with the same FX chains, items time-windowed.
  5. Open each, dial in any per-song mix tweaks, render that song's stereo bounce.

Render output dial-in (per LoudMax + Youlean)

Two-pass workflow for hitting both -14 LUFS streaming and -9 LUFS club targets:

  1. Set LoudMax Threshold for streaming (-2 to -3 dB), render once → mixdowns/streaming/$project-$region.wav.
  2. Lower LoudMax Threshold to -4 or -5 dB for club, render again → mixdowns/club/$project-$region.wav.
  3. Don't run two limiters in series — same chain, just different drive amounts. Compare both bounces in Youlean to verify integrated LUFS hits target.

8 · Live multitrack gotchas

Cymbal bleed on tom mics

The #1 thing that breaks between gigs. Tom mics pick up 80% bleed. Different rooms = different bleed levels. Always check tom gates per-gig. Symptoms:

  • Random "ghost tom" hits in the mix → gate threshold too low; raise by 3-5 dB
  • Tom hits cut off too early → hold time too short; raise to 350-500 ms
  • Cymbal triggers gate occasionally → hysteresis too small; raise to 12-15 dB

Vocal mic monitor bleed pumps comp

Stage monitors leaking into vocal mic make compressor "breathe" between phrases. Symptoms: comp release is audible, vocal sounds "hollow" between words.

  • Raise vocal gate threshold by 3-5 dB so it closes between phrases.
  • Shorten ReaComp release to 50 ms so it recovers faster.
  • Sidechain HP filter on the vocal comp at 200 Hz so kick bleed doesn't drive it.

OH HPF: 80 vs 150 Hz vs 200 Hz

The OH HPF is a compromise. Rules of thumb:

  • 80 Hz — kick mic failed; OH IS your kick source.
  • 120-150 Hz — normal: OH gives cymbal body + room glue, kick mic on Ch 10 carries the punch.
  • 180-200 Hz — band is loud + kick is well-mic'd; aggressive HPF keeps low-end out of the OH summation.

EAD + acoustic kick phase cancellation

Yamaha EAD trigger has small latency vs close kick mic. If they sum constructively, all good. If they cancel, low-end disappears when both are unmuted.

  • Solo Kick (Ch 10), then unmute EAD (Ch 8-9). If low-end DROPS, flip phase on Kick: track header → click φ button OR Track menu → Toggle phase.
  • If still off, nudge Ch 10 timing forward by 5-15 samples (right-click item → Item properties → Length+Position).

Don't compress the Music bus

Backing tracks are already mastered to -8 LUFS or so. Compressing them creates audible pumping against the live drum hits. Even subtle 1-2 dB GR is audible. Just gain-match the bus fader.

Reverb on drums = sounds fake

Live recordings already have natural room ambience baked into OH mics. Adding reverb to the drum bus makes it sound like a studio multitrack pretending to be live. Don't.

Per-song bounces inherit master chain unchanged

Set the master mastering chain ONCE at the start of the post-prod session. Don't re-tune it per song. The shared mastering chain ensures all songs sound consistent — that's the whole point.

Headroom: leave -6 dB on master fader

Even with LoudMax catching peaks, leave the master fader at -6 dB to give yourself room for last-second EQ boosts without clipping. The mastering chain wants headroom on the way in.
